==========================================================================
K210双功能程序使用说明
==========================================================================

一、功能说明
==========================================================================

功能1：颜色识别模式
- 识别红、绿、黄三种颜色小球
- 实时计数每种颜色的数量
- 在LCD上显示识别结果

功能2：紫色跟随模式
- 识别紫色矩形
- 跟随紫色矩形移动
- 发送坐标和大小数据给STM32

二、通讯协议
==========================================================================

1. 颜色识别模式数据格式：
   $C + R + G + Y + #
   示例：$C213# 表示红色2个，绿色1个，黄色3个

2. 跟随模式数据格式：
   $F + XXX + YYY + WWW + HHH + #
   示例：$F160120050060# 表示X=160, Y=120, W=50, H=60

3. 模式切换命令（STM32 -> K210）：
   $M1# - 切换到颜色识别模式
   $M2# - 切换到跟随模式

三、文件说明
==========================================================================

K210端：
- k210_dual_function.py  - K210主程序（双功能）
- color_ball_detect.py   - 单独的颜色识别程序

STM32端：
- app_k210_dual.h        - 头文件
- app_k210_dual.c        - 实现文件

四、接线说明
==========================================================================

K210 <-> STM32：
- K210 TX (P9)  -> STM32 PA3 (USART2 RX)
- K210 RX (P10) -> STM32 PA2 (USART2 TX)
- K210 GND      -> STM32 GND

五、使用步骤
==========================================================================

1. 将 k210_dual_function.py 烧录到K210

2. 将 app_k210_dual.h 和 app_k210_dual.c 添加到STM32工程

3. 在STM32的 main.c 中添加：
   #include "app_k210_dual.h"

4. 在串口中断处理函数中添加：
   void USART2_IRQHandler(void)
   {
       ...
       Deal_K210_Dual(Rx2_Temp);  // 添加这行
       ...
   }

5. 在主循环中添加处理代码：
   if(mode == K210_Follow)
   {
       K210_Color_Count_Process();  // 颜色识别处理
       K210_Follow_Process();       // 跟随处理
   }

六、阈值调整
==========================================================================

如果颜色识别不准确，需要调整K210程序中的阈值：

# 红色阈值
red_threshold = (30, 70, 40, 80, 20, 70)

# 绿色阈值
green_threshold = (30, 70, -70, -20, 10, 60)

# 黄色阈值
yellow_threshold = (50, 90, -20, 20, 50, 90)

# 紫色阈值
purple_threshold = (20, 70, 20, 80, -80, -20)

可以使用K210 IDE的阈值调整工具来获取准确的阈值。

七、注意事项
==========================================================================

1. 确保K210和STM32的波特率一致（115200）

2. 颜色阈值需要根据实际光照环境调整

3. 紫色矩形应该有足够大的面积（建议大于300像素）

4. 如果需要切换模式，可以通过蓝牙发送命令给STM32，
   再由STM32发送模式切换命令给K210
